# FAQ 机器人 PRD（v2）

## 📌 项目概述

| 项目 | 内容 |
|------|------|
| **项目名称** | 咖啡知识 FAQ 智能助手 |
| **核心价值** | 7×24 小时自动回答用户咖啡相关问题 |
| **目标用户** | 咖啡客户、潜在买家、批发商 |
| **使用渠道** | 飞书（个人版，小群测试） |
| **技术栈** | Python + Ollama + FAISS + Socket.IO |

---

## 🔄 系统架构

```
┌─────────────────────────────────────────────────────┐
│                    OpenClaw（知识库管理员）             │
│  • 维护 coffee 知识库（新增/修改/删除）                  │
│  • 分析用户反馈，更新知识库                              │
└─────────────────────────────────────────────────────┘
                         ↑ 知识库更新
                         │ 反馈问题
┌─────────────────────────────────────────────────────┐
│              FAQ 机器人（RAG + 飞书 WebSocket）         │
│  • 接收用户问题（飞书 WebSocket）                      │
│  • 检索 coffee 知识库生成回答                          │
│  • 收集用户反馈发送给 OpenClaw                         │
└─────────────────────────────────────────────────────┘
                         ↑ 问答服务
                         │
┌─────────────────────────────────────────────────────┐
│                    用户（飞书群）                      │
│  • @机器人 提问                                       │
└─────────────────────────────────────────────────────┘
```

### RAG 检索流程

```
用户提问 → 生成 query 向量 → FAISS 召回 Top-K 片段 → 组装 Prompt → Ollama Qwen 生成回答
```

### 索引更新流程

```
用户首次提问 → 检测索引过期/不存在 → 扫描 coffee/*.md → 重建 FAISS 索引 → 继续查询
                                    ↑
                          每天定时任务触发
```

## 🤖 机器人职责

| 需求ID | 需求描述 | 优先级 |
|--------|---------|--------|
| BR-01 | **接收问题** - 用户 @机器人 或发消息 | P0 |
| BR-02 | **检索回答** - 从 coffee 知识库检索相关内容，生成回答 | P0 |
| BR-03 | **来源标注** - 回答注明知识库来源（如：来自 耶加雪菲.md） | P1 |
| BR-04 | **收集反馈** - 无法回答的问题，反馈给 OpenClaw | P0 |
| BR-05 | **不知道就说** - 知识库没有的，回复"暂无此信息"并反馈 | P0 |

---

## 📤 反馈机制

| 反馈类型 | 处理方式 |
|----------|----------|
| 用户问了一个知识库没有的问题 | 记录问题 → 反馈给 OpenClaw → OpenClaw 更新知识库 |
| 用户指出回答有误 | 记录反馈 → 反馈给 OpenClaw → OpenClaw 修正 |

**反馈格式**（发给 OpenClaw）：
```markdown
## 用户反馈
- 时间：2026-05-26
- 用户问题：xxx
- 机器人回复：xxx
- 反馈类型：新增问题 / 回答有误
```
